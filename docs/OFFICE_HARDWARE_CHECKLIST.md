## F1 — Контроллер (репозиторий: шаблон; онсайт: F1-O1…O3)

### Подключение и порт / Connection and port

1. Контроллер подключён согласно регламенту стенда (кабель, питание, заземление — по чеклисту инженера).
2. **Порт и скорость** согласованы с эталоном: см. **`viwa_electron/docs/SERIAL_PORTS_IMPLEMENTATION.md`** и **§2** основного ТЗ (`TZ_WIVA_ANDROID_FROM_ELECTRON.md` — строка таблицы «Протокол контроллера»).
3. Значения в приложении (путь устройства, baud) проверены против `ControllerPortSettings` и настроек сервисного меню (если вынесены).

| Шаг | Выполнено ☐ | Заметки |
|-----|-------------|---------|
| Подключение физическое | ☐ | |
| Порт/скорость сверены с SERIAL_PORTS + ТЗ §2 | ☐ | |

### Мок ↔ реал / Mock vs real path

- **Ключ:** `JsonStoreKeys.USE_MOCK_CONTROLLER` (в хранилище: `useMockController`).
- **Где менять:** сервисное меню Wiva (группа/вкладка по структуре из ТЗ §2 — «Сервисное меню»).
- **Для стенда F1 (проверка железа):** мок контроллера **должен быть выключён** (как в ТЗ §3: на стенде проверяется реальная цепочка; понимание ограничений транспорта — см. блок «Блокер» ниже).
- Детали кода: `ControllerPortSettings.kt`, `ControllerModule.kt`, `DelegatingControllerGateway`.

### Регрессия vs мок (happy-path) / Regression vs mock

Сценарий согласован с **§3** ТЗ и эмулятором (мок в Electron: `MockControllerConnection.ts` и тесты рядом — для ожиданий по смыслу).

**Оплата в этом прогоне:** реальный терминал (**раздел F2 ниже**) или мок оплаты — по договорённости стенда; зафиксируйте в колонке «Заметки».

### Блокер стенда / отдельная задача (§4.1 ТЗ)

Если **реальный байтовый обмен** с контроллером на Android **невозможен** (транспорт всё ещё заглушка: лог TX без чтения с порта) — **не скрывать** в отчёте стенда. Оформить как:

- **блокер стенда** для критериев, требующих реального RX; и/или  
- **отдельная задача** по реализации транспорта / USB-serial после согласования с владельцем продукта (**§4.1** `TZ_WIVA_ANDROID_FROM_ELECTRON.md`).
